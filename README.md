# workday-mcp-template

This project provides a starter template for building MCP servers that integrate with Workday-based workflows.

The repository focuses on:

1. Reusable MCP server infrastructure (config-driven **bootstrap**: load config → init services → create server → register tools → attach stdio/HTTP transport). See [docs/architecture.md](docs/architecture.md#mcp-server-bootstrap).

2. Workday-aligned tool schemas generated by OpenAPI specs.

3. (Maybe) enterprise-safe policy and validation layers

The goal is to allow developers and Workday customers to quickly build AI agent tools that interact with workforce systems.

This project is an independent developer template built using publicly available Workday documentation and announcements.
It is not affiliated with or endorsed by Workday.

Minimum setup you should expect from consumers:

- API client registration in the tenant, Workday admin/integration admin involvement
- Scopes/permissions granted on API client
- Tenant/base URL and OAuth endpoints supplied
- An integration user or authorized Workday account with the needed permissions
- Look up OpenAPI specs for Workday tools you want to use. Normally avaiable to partners or customers on Workday Community or Workday Developer Site.
